## 本地环境顺序

| 步骤 | 命令 | 说明 |
|------|------|------|
| 1 | `pnpm docker:reset` | L1a seed + L1b key sync（见 [启动优化 §2.1](../本地开发-启动优化.md#21-pnpm-dockerreset-阶段链)） |
| 2 | `pnpm start` / `start:lite` | 只保证 L0；不 wait readiness |
| 3 | （可选）`curl …/api/dev/readiness` | 查 L1b；E2E 用 `pnpm verify gate` |

`setup-dev-mock-channel.sh` 对 channel 为 **best-effort**；分组注册由 Backend `EnsureGroup`（FIX-INT-001）为主。

---

## FIX-DEV-001 — 模拟消耗报「无权访问 dept-dept-3 分组」

1. 确认 Backend 已含 FIX-INT-001（`TrySyncCreate` → `EnsureGroup`）
2. 重跑 `pnpm docker:reset && pnpm start`，或单独 `setup-dev-mock-channel.sh`
3. 用 **plk-1** 测 `local-test-model`（dev channel 在 `dept-dept-3`）

---

## FIX-DEV-002 — 分组 OK 仍 `no access to model local-test-model`

1. seed 白名单变更后重跑 `pnpm docker:reset`（`dev-bootstrap` 会同步重建 token 的 `model_limits`）
2. 或对用户 Key 执行 Update / Rotate 触发 `SyncUpdatePlatformKey`
3. [FIX-SEED-003](../../apps/backend/seed/FIXES.md#fix-seed-003) — 确认白名单含 model **1**
4. 推荐 Key：`plk-1`（`seed/FIXES.md` 对齐表）

---

## FIX-DEV-003 — Gateway 过、NewAPI 挂（dev 豁免误解）

| | |
|--|--|
| **原因** | `local-test-model` 在 local 跳过 Gateway 白名单，NewAPI 仍校验 `group` + `model_limits` |
| **典型** | 选 `plk-1b`（白名单无 model 1） |
| **处理** | 以 NewAPI 为准；测试通路用 plk-1 |

## FIX-DEV-005 — `model_price_error`（local-test-model 价格未配置）

| | |
|--|--|
| **原因** | Channel/分组 OK，但 NewAPI `ModelRatio` 未含 `local-test-model` |
| **处理** | 重跑 `apps/newapi/scripts/setup-dev-mock-channel.sh`（会写 ModelRatio/CompletionRatio，对齐 gpt-4o-mini） |
| **永久** | `pnpm docker:reset` → bootstrap 已含该步骤 |

---

## FIX-DEV-006 — `upstream_error` / `not found`（路径 /v1 重复）

| | |
|--|--|
| **原因** | Channel `base_url` 带 `/v1`，NewAPI 再拼 `/v1/...` → 双重路径 404 |
| **处理** | `setup-dev-mock-channel.sh` 经 `verify_http_origin`（对齐 Go `baseurl.Origin`）规范化；重跑即可 |
| **校验** | Gateway 模拟消耗应 HTTP 200 |

## 成功路径（验收）

```bash
pnpm docker:reset && pnpm start
curl -s localhost:8080/api/dev/readiness   # 期望 200
# 可选冒烟
pnpm verify:gate
```

1. 登录 `admin@example.com` / `demo1234`
2. Popup → **张三-开发调试（plk-1）**
3. Gateway **200**；audit 出现 `local-test-model`

仍失败：按 [Backend-NewAPI集成修复.md §诊断](../Backend-NewAPI集成修复.md#诊断) 查 `UserUsableGroups` 与 token `group` / `model_limits`。
