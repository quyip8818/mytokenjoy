## 本地环境顺序

| 步骤 | 命令 | 跳过后果 |
|------|------|----------|
| 1 | `pnpm docker:reset` | 旧 model id、旧 NewAPI token |
| 2 | （内含）`bootstrap-local-after-reset.sh` | 无 Admin Token |
| 3 | （内含）`setup-dev-mock-channel.sh` | [FIX-INT-001](../Backend-NewAPI集成修复.md#fix-int-001) |
| 4 | `pnpm start` | 无 demo seed / Worker |
| 5 | 等 Worker sync seed keys | 无 mapping / bearer |

`bootstrap` 对 channel 为 **best-effort**（失败仅 WARN）→ 易出现分组未注册。

## FIX-DEV-001 — 模拟消耗报「无权访问 dept-dept-3 分组」

1. 查 [FIX-INT-001](../Backend-NewAPI集成修复.md#fix-int-001)（主因：NewAPI 未注册 group）
2. 重跑 `apps/newapi/scripts/setup-dev-mock-channel.sh`
3. 用 **plk-1**，勿用跨 `dept-dept-5` 的项目 Key 测 `local-test-model` channel（channel 仅在 `dept-dept-3`）

---

## FIX-DEV-002 — 分组 OK 仍 `no access to model local-test-model`

1. [FIX-INT-003](../Backend-NewAPI集成修复.md#fix-int-003) — rotate Key 刷新 `model_limits`
2. [FIX-SEED-003](../../apps/backend/seed/FIXES.md#fix-seed-003) — 确认白名单含 model **1**（非旧文档中的 9）
3. 推荐 Key：`plk-1`（`seed/FIXES.md` 对齐表）

---

## FIX-DEV-003 — Gateway 过、NewAPI 挂（dev 豁免误解）

| | |
|--|--|
| **原因** | `local-test-model` 在 local 跳过 Gateway 白名单，NewAPI 仍校验 `group` + `model_limits` |
| **典型** | 选 `plk-1b`（白名单无 model 1） |
| **处理** | 以 NewAPI 为准；测试通路用 plk-1 |

## 成功路径（验收）

```bash
pnpm docker:reset && pnpm start
```

1. 登录 `admin@example.com` / `demo1234`
2. Popup → **张三-开发调试（plk-1）**
3. Gateway **200**；audit 出现 `local-test-model`

仍失败：按 [Backend-NewAPI集成修复.md §诊断](../Backend-NewAPI集成修复.md#诊断) 查 `UserUsableGroups` 与 token `group` / `model_limits`。
