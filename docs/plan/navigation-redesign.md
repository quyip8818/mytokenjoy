# 侧边导航栏重组方案

## 现状

```
组织
  数据源
  组织架构
  角色管理
预算
  预算管理
  预警规则
模型管理
  模型列表
  模型配置
Key 中心
  我的 Key
  审批中心
  Key 管理
  供应商 Key
数据中心
  成本看板
  用量分析
财务管理
  钱包管理
审计
  操作审计
  调用日志
───────────────
个人中心（独立布局，不在侧栏内）
  工作台
  我的 Key
  使用记录
  通知偏好
  账户设置
  登录活动
───────────────
独立页面（无导航栏）
  登录页
  邀请激活页
```

**7 个分组 · 16 个管理页 · 6 个个人页 · 两套布局 · 2 个独立页面**

### 问题

1. 高频功能（看板、审批）排在低频功能（组织）后面
2. "Key 中心"混合 4 种不同性质的操作（个人自助 / 流程审批 / 全局管控 / 基础设施配置）
3. "财务管理"只有 1 个子项独占一个分组
4. 分组用抽象动词命名（"Key 中心""数据中心"），需要二次猜测
5. 管理员看自己的 Key 或改设置要切换到独立布局
6. 个人中心和管理后台存在重复入口（"我的 Key"出现两次）

---

## 设计方向

- **保留分组结构**，分组支持展开/折叠
- **个人中心合并进侧栏**，取消独立布局
- 重新划分分组归属，调整排序

以下三个方案在分组划分方式上有差异。

---

## 方案 A：按职能域分组

分组逻辑：每个组对应一个管理职能。

```
数据看板                         ← 展开
  成本看板
  用量分析

凭证管理                         ← 展开
  Key 管理
  审批中心
  供应商 Key

模型管理                         ← 展开
  模型列表
  模型配置

预算与财务                       ← 折叠
  预算管理
  预警规则
  钱包管理

组织与权限                       ← 折叠
  数据源
  组织架构
  角色管理

审计                             ← 折叠
  操作审计
  调用日志

════════════════════（底部固定）
👤 用户名
  我的 Key
  我的用量
  设置                           ← 账户设置 + 登录活动 + 通知偏好合并
```

**6 个分组 · 15 个管理子项 · 底部 3 个个人项**

要点：
- "供应商 Key"归入凭证管理（和其他 Key 相关操作同组）
- "钱包管理"归入预算与财务（都是"钱的事"）
- "我的 Key"从管理区移到底部个人区
- 高频组（数据看板、凭证管理）默认展开，低频组默认折叠

---

## 方案 B：按操作对象分组

分组逻辑：每个组对应一个核心实体/对象。

```
概览                             ← 展开
  成本看板
  用量分析
  调用日志

API Keys                         ← 展开
  Key 管理
  审批中心

模型与供应商                     ← 展开
  模型列表
  模型配置
  供应商 Key

费控                             ← 折叠
  预算管理
  预警规则
  钱包管理

系统设置                         ← 折叠
  组织架构
  角色管理
  数据源
  操作审计

════════════════════（底部固定）
👤 用户名
  我的 Key
  我的用量
  设置
```

**5 个分组 · 15 个管理子项 · 底部 3 个个人项**

要点：
- "调用日志"归入概览（都是"看数据"的场景）
- "供应商 Key"归入模型与供应商（配置模型时顺手配供应商）
- "操作审计"归入系统设置（都是低频管理操作）
- 分组更少（5 个），每组 3-4 项
- 命名用实体名词（API Keys、模型与供应商）

---

## 方案 C：按工作流分组

分组逻辑：按管理员日常工作流的顺序排列。

```
日常运营                         ← 展开
  成本看板
  用量分析
  审批中心
  调用日志

资源管理                         ← 展开
  Key 管理
  模型列表
  模型配置
  供应商 Key

规则与额度                       ← 折叠
  预算管理
  预警规则
  钱包管理

基础配置                         ← 折叠
  组织架构
  角色管理
  数据源
  操作审计

════════════════════（底部固定）
👤 用户名
  我的 Key
  我的用量
  设置
```

**4 个分组 · 15 个管理子项 · 底部 3 个个人项**

要点：
- 把管理员每天要做的事（看板、审批、日志）放在"日常运营"
- 把管理员要"管的东西"（Key、模型、供应商）放在"资源管理"
- 分组最少（4 个），但每组子项较多（4 项）
- 高频操作集中在前两组，低频操作集中在后两组

## 方案对比

| 维度 | A 按职能域 | B 按操作对象 | C 按工作流 |
|------|:---------:|:-----------:|:----------:|
| 分组数 | 6 | 5 | 4 |
| 每组平均子项 | 2.5 | 3 | 3.75 |
| 高频功能定位速度 | 好（前两组） | 好（前两组） | 最好（第一组全覆盖） |
| 分组命名直觉度 | 中（仍有"凭证管理"这类抽象名） | 好（名词实体） | 好（日常运营/资源管理好懂） |
| 组内逻辑内聚 | 高（同一职能） | 高（同一对象） | 中（"日常运营"含审批+看板+日志，关联弱） |
| 新功能归属清晰度 | 高（新功能必属于某个职能） | 高 | 中（有些功能不确定是"运营"还是"管理"） |
| 可扩展性 | 好 | 好 | 中（组太少，膨胀后单组过长） |
| 与现状差异大小 | 小（分组思路接近） | 中 | 大（完全重组） |

## 共同设计点（三个方案通用）

### 个人区域

底部固定，不随主导航滚动：
- 用户头像 + 姓名 + 部门（点击展开：切换组织 / 退出登录）
- 我的 Key
- 我的用量（原"使用记录"）
- 设置（合并：账户设置 + 登录活动 + 通知偏好）

### 取消的内容

- 原"工作台"页面取消（用量摘要并入"概览/成本看板"按角色展示，Key 状态并入"我的 Key"顶部）
- 独立个人中心布局取消，统一为单一侧栏

### 折叠/展开行为

- 高频组默认展开，低频组默认折叠
- 用户手动展开/折叠的状态持久化（记住偏好）
- 折叠时只显示分组名，不显示子项

### 普通成员视图

权限控制后，成员只看到：
- 概览/成本看板（个人视角数据）
- 调用日志（仅自己的）
- 底部个人区 3 项

管理分组全部隐藏。

### 独立页面（不变）

- 登录页
- 邀请激活页
