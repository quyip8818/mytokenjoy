## 1. 产品概述

### 1.1 产品定位

TokenJoy 面向企业内部 AI 落地场景，解决多团队共用 LLM 资源时的三大核心问题：

| 问题域 | 解决方案 |
|--------|----------|
| 💰 预算管控 | 逐级预算分配 + 超限阻断 + 预警通知 + 自然月重置 |
| 🔒 安全合规 | 模型白名单 + 操作审计 + 调用日志 + 内容留存 |
| 📊 可观测性 | 成本看板 + 用量分析 + 部门下钻 + CSV 导出 |

### 1.2 产品形态

- **SaaS 多租户**：平台托管多家企业，数据按企业行级隔离
- **私有化部署**：同一制品，单企业模式

### 1.3 目标用户与角色

| 角色 | 范围 | 核心诉求 |
|------|------|----------|
| 超级管理员 | 全企业 | 组织初始化、预算分配、模型管控、合规审计 |
| 组织管理员 | 全企业 | 团队管理、预算审批、Key 管理 |
| 普通成员 | 个人 | 申请 Key、调用 API、查看用量 |
| 只读审计员 | 全企业 | 审计日志查看（只读） |
| API 调用者 | 个人 | 纯 API 调用 |

### 1.4 业务阶段

```
P1 平台初始化 → P2 资源管控配置 → P3 成员接入与调用 → P4 运营与合规
```

- P1：接入第三方平台、导入组织、配置角色
- P2：分配预算、设置白名单、配置预警
- P3：审批 Key、成员自主管理、API 调用
- P4：成本分析、用量监控、审计追踪

## 2. 组织管理

### 2.1 数据源配置

**入口**：侧边栏 > 组织 > 数据源
**权限**：数据源配置

#### 功能描述

管理员配置第三方平台（飞书/钉钉/企微）开发者凭证，用于连接组织架构数据。

#### 操作流程

1. 选择平台（飞书/钉钉/企微）
2. 填写平台凭证
3. 测试连接
4. 保存凭证
5. 执行全量导入（可选）
6. 配置定时同步（可选）

#### 各平台凭证字段

| 平台 | 必填字段 |
|------|----------|
| 飞书 | App ID, App Secret |
| 钉钉 | CorpID, AppKey, AppSecret |
| 企微 | CorpID, Secret, AgentID |

#### 业务规则

- 一家企业只能接入一个第三方平台
- 保存前必须通过测试连接验证
- 凭证更换不影响已导入数据
- 切换平台需二次确认（清除当前凭证配置）

#### 全量导入

- 从第三方平台全量拉取部门树 + 成员信息
- 重复导入：增量合并（新增加入、已有不变）
- 导入后成员默认角色："普通成员"
- 部分失败时展示失败详情表格（姓名、工号、原因），支持重试

#### 定时同步策略

| 配置项 | 选项 | 默认值 |
|--------|------|--------|
| 同步频率 | 6 / 12 / 24 小时 | 12 小时 |
| 每日开始时间 | 自定义 | 02:00 |
| 成员删除保护阈值 | 自定义 | 10 人 |
| 部门删除保护阈值 | 自定义 | 5 个 |

同步规则：
- 第三方导入数据：以第三方为准（新增加入、移除软删除、改名跟随）
- 本地手动添加数据：不受同步影响
- 超过删除保护阈值：立即终止同步并通知超管
- 同步日志持久化：展示每次同步的时间、类型（定时/手动）、结果

### 2.2 组织架构

**入口**：侧边栏 > 组织 > 组织架构
**权限**：组织架构管理、成员管理

#### 功能描述

管理企业部门树和成员，支持第三方导入与手动管理并存。

#### 部门管理

- 左侧部门树面板（多级嵌套，支持搜索）
- 新建部门（指定父节点）
- 编辑部门（改名）
- 删除部门（需先清空子部门和成员）

#### 成员管理

- 右侧成员列表（按选中部门过滤）
- 添加成员：手动填写（姓名、手机号等）
- 邀请成员：发送邀请链接，成员自行激活
- 编辑 / 停用 / 启用 / 删除
- 搜索、按状态筛选

#### 批量操作

- 转移部门
- 批量启用 / 停用
- 批量删除

#### 成员状态

```
[创建] → 启用
[邀请] → 未激活 → 启用
启用 ⇄ 停用
停用 / 启用 → 删除
```

#### 业务规则

- 停用成员 → 其 Platform Key 同步失效
- 筛选支持：仅查看直属 / 全部成员
- 未激活成员需先激活才可使用
- 待激活成员有专用 banner 提示

### 2.3 角色管理

**入口**：侧边栏 > 组织 > 角色管理
**权限**：角色管理

#### 功能描述

管理系统角色并分配给成员，控制操作权限。

#### 预设角色（不可删除/修改）

| 角色 | 定位 |
|------|------|
| 超级管理员 | 全局权限，企业 IT/AI 管理者 |
| 组织管理员 | 与超管权限一致，团队负责人 |
| 普通成员 | 默认保底角色，不可移除 |
| 只读审计员 | 只读审计视角 |
| API 调用者 | 纯调用角色 |

#### 自定义角色

- 创建：命名 + 勾选权限点
- 编辑：修改名称或权限集
- 删除：需确认（提示影响成员数）

#### 角色成员管理

- 查看角色下的成员列表
- 添加成员到角色（搜索选择）
- 从角色中移除成员

#### 业务规则

- 一个成员可拥有多个角色（权限取并集）
- 普通成员为保底角色，所有人默认拥有且不可移除
- 角色变更即时生效
- 权限集由后端动态下发

## 3. 预算管控

### 3.1 预算管理

**入口**：侧边栏 > 预算 > 预算管理
**权限**：查看预算、分配预算

#### 功能描述

逐级分配预算额度，实现各级自主成本管控。预算按组织树从上至下分配。

#### 预算分配模型

```
Company 总预算
├── 部门 A（分配额度）
│   ├── 子部门 A1（分配额度）
│   │   ├── 成员平均额度
│   │   └── 预留池（共享额度）
│   └── Budget Group（虚拟项目组）
├── 部门 B（分配额度）
└── 未分配余额
```

#### 页面结构

- 左侧：预算树面板（含月份切换导航）
- 右侧：选中节点详情
  - 团队详情：已分配/未分配/已使用、子节点列表
  - 项目组详情：成员列表、独立额度
  - 成员额度：个人额度、已用、申请记录

#### Budget Group（虚拟项目组）

- 独立于组织树的预算分组
- 预算从创建团队额度中扣除
- 组内调用走 Group 独立额度
- 拥有独立 API Key
- 支持：创建 / 编辑 / 删除 / 添加成员

#### 成员额度管理

- TL 配置"平均额度/人"
- 预留池为共享额度
- 成员额度用完需申请追加（走审批流）
- TL 审批后从预留池扣除

#### 待审批

- 预算页面内嵌待审批抽屉
- 展示等待当前用户审批的额度申请
- 通过 / 拒绝操作

#### 业务规则

- 子节点预算之和 + 预留池 ≤ 父节点总额度（不允许超卖）
- 预算单位：人民币（元）
- 预算周期：自然月，月初自动重置（配置不变，已用清零）
- 超限策略：额度用完直接阻断 API 请求

---

### 3.2 预警规则

**入口**：侧边栏 > 预算 > 预警规则
**权限**：预警策略配置

#### 功能描述

配置预算消耗预警阈值和通知规则。

#### 预警规则配置

每条规则包含：
- 目标类型：团队 / 项目组
- 目标名称
- 阈值列表（多级，如 80%、90%、100%）
- 通知角色（选择需通知的角色）
- 启用/停用开关

#### 阈值展示

- 按严重程度颜色编码
- Badge 形式展示在列表中

#### 操作

- 创建规则
- 编辑规则
- 删除规则
- 启用/停用切换

#### 业务规则

- 预警阈值可添加多个（多级预警）
- 达到 100% 时阻断请求
- 通知方式：IM + 邮箱
- 通知发送失败不影响阻断逻辑

## 4. 模型管理

### 4.1 模型列表

**入口**：侧边栏 > 模型管理 > 模型列表
**权限**：模型管理、查看模型

#### 功能描述

管理平台可用的 LLM 模型（系统模型 + 企业自定义模型）。

#### 模型信息

每个模型包含：供应商、模型名称、显示名称、输入/输出单价（元/千token）、最大上下文长度、能力标签、启用状态。

#### 操作

- 查看模型列表（含供应商、价格、上下文长度）
- 添加自定义模型（名称、接口地址、密钥、价格）
- 启用 / 停用模型

---

### 4.2 模型白名单（路由规则）

**入口**：侧边栏 > 模型管理 > 模型白名单
**权限**：白名单配置

#### 功能描述

按部门配置可用模型白名单，控制各部门的模型使用范围。

#### 继承逻辑

- 默认：继承父级白名单
- 可切换为"自定义"：从父级白名单中选择子集
- 只能缩小不能扩大
- 父级缩小 → 子级自动同步缩小

#### 路由规则表

每条规则：
- 组织节点（部门名）
- 模式：继承 / 自定义
- 允许的模型列表
- 默认模型
- 备选模型（fallback）

#### 业务规则

- 控制粒度：部门级别
- API 调用时指定模型不在白名单内 → 返回错误
- 未指定模型 → 返回错误

## 5. Key 中心

### 5.1 我的 Key

**入口**：侧边栏 > Key 中心 > 我的 Key
**权限**：个人 Key 自管理

#### 功能描述

成员自主管理个人 Platform Key，在个人额度内自由创建和使用。

#### 页面结构

- 顶部：额度统计卡片（总额度 / 已用 / 剩余）
- 主体：Key 列表

#### Key 操作

| 操作 | 说明 |
|------|------|
| 创建 | 选绑定模型 + 分配额度（不超个人剩余） |
| 编辑 | 调整模型绑定、额度分配 |
| 轮转 | 重新生成密钥（旧 Key 立即失效，配置继承） |
| 启用/停用 | 切换状态 |
| 删除 | 释放未用额度回个人池 |
| 申请额度 | 额度不足时发起追加申请（走审批流） |

#### Key 属性

- 名称：用途描述
- 密钥：脱敏展示（支持复制完整值）
- 状态：正常 / 停用 / 过期 / 异常
- 额度：分配额度
- 已用：已消耗额度
- 绑定模型：允许调用的模型列表
- 过期时间（可选）

#### 业务规则

- 无需审批，个人额度内自主创建
- 一个成员可有多个 Key，各 Key 独立计费
- Key 额度用完 → 该 Key 不可用（其他 Key 不受影响）
- Key 额度之和不能超出个人总额度
- 删除 Key 释放未用部分回个人池

---

### 5.2 审批中心

**入口**：侧边栏 > Key 中心 > 审批中心
**权限**：审批额度、个人审批

#### 功能描述

统一管理 Key 申请和额度追加审批。

#### 页面结构

三个 Tab：
- **待我审批**：当前用户需审批的申请
- **我的申请**：当前用户发起的申请
- **全部**：所有审批记录

#### 审批类型

| 类型 | 申请信息 | 通过后系统动作 |
|------|----------|----------------|
| Key 申请 | 申请人、理由、需要的模型 | 自动创建 Platform Key |
| 额度追加 | 申请人、当前/已用额度、申请金额 | 从预留池扣除追加到个人 |

#### 操作

- 提交新申请
- 通过审批
- 拒绝审批（可填理由）
- 待审批数量 Badge 提示

#### 业务规则

- 单人审批（直属 TL）
- 无超时自动处理
- 审批结果即时生效
- 预留池不足时阻止通过

## 6. 数据中心

### 6.1 成本看板

**入口**：侧边栏 > 数据中心 > 成本看板
**权限**：成本看板

#### 功能描述

全局展示 LLM 使用成本、趋势和效率指标，支撑预算决策。

#### 页面结构

**指标卡片（4 项）**：
| 指标 | 说明 |
|------|------|
| 总花费 | 当前周期总成本 + 环比% |
| 平均单次成本 | 总花费/总请求数 + 环比% |
| 人均成本 | 总花费/活跃成员数 + 环比% |
| 总调用次数 | 总请求数 + 环比% |

**趋势图表**：
- 花费趋势折线图（粒度：小时/天/周/月）
- 部门花费占比饼图

**明细表格**：
- 部门花费列表（支持下钻）
- 下钻路径：部门 → 子部门 → 成员
- Top 消费者排行

#### 时间维度

- 本月 / 上月 / 近 7 天 / 自定义范围
- 环比基于同等时间长度对比

---

### 6.2 用量分析

**入口**：侧边栏 > 数据中心 > 用量分析
**权限**：用量分析

#### 功能描述

按团队和模型维度分析 API 调用用量。

#### 页面结构

**团队用量表格**：
| 列 | 说明 |
|----|------|
| 部门名称 | 团队名 |
| 额度 | 分配总额度 |
| 已用 | 已消耗额度（含进度条） |
| 成员数 | 部门人数 |
| 最常用模型 | Top 1 模型 |

**模型用量分布**：
- 各模型的请求数、Token 数、花费、占比

#### 用量时序查询

支持高级时序查询：
- 粒度：分钟 / 小时 / 天
- 分组：无 / 按部门 / 按成员 / 按模型
- 时间范围自定义
- 数据来源标注（buckets / ledger）

## 7. 审计合规

### 7.1 操作审计

**入口**：侧边栏 > 审计 > 操作审计
**权限**：`audit:read`

#### 功能描述

记录所有管理操作，支持查询和导出。

#### 记录内容

每条操作日志包含：操作类型、操作人、操作目标、详情、IP 地址、时间。

#### 操作类型

- 创建 Key / 停用 Key / 轮转 Key
- 预算变更 / 预算审批
- 权限变更 / 角色分配
- 白名单变更
- 添加成员 / 移除成员
- 组织架构变更

#### 筛选与导出

- 按操作类型筛选
- 按时间范围筛选
- 按操作人筛选
- 关键词搜索
- 分页
- CSV 导出

---

### 7.2 调用日志

**入口**：侧边栏 > 审计 > 调用日志
**权限**：`audit:read`

#### 功能描述

记录所有 LLM API 调用详情，支持查询和内容留存配置。

#### 记录内容

每条调用日志包含：调用人/应用名、调用者类型、使用模型、供应商、输入/输出 Token 数、响应延迟、状态（成功/失败）、费用、内容预览片段。

#### 功能

- 可展开行查看 prompt/response 预览
- 筛选：状态、模型、调用人、时间范围、关键词
- 分页
- CSV 导出
- 内容留存开关（可配置是否记录输入输出内容）

#### 业务规则

- 审计记录不可篡改、不可删除
- 只读审计员只能查看，无修改权限
- 内容留存开关在审计设置中配置

---

## 8. 财务管理

### 8.1 钱包管理

**入口**：侧边栏 > 财务管理 > 钱包管理
**权限**：`billing:read`, `billing:recharge`

#### 功能描述

管理企业钱包余额和充值记录。

#### 页面结构

**统计卡片**：
- 当前余额
- 累计消费
- 累计充值 / 总请求数

**充值操作**：

#### 业务规则

- 余额不足时 API 调用被阻断
- 充值记录持久化

## 10. 系统基础

### 10.1 认证

- 登录方式：邮箱 + 密码
- 401 未认证时自动跳转到登录页

### 10.2 权限控制

- 路由级：无权限的页面不可访问，自动跳转到首个可访问页面
- 操作级：无权限的按钮/操作不展示或禁用
- 数据级：后端按当前用户权限鉴权

### 10.3 API 调用链路

```
成员 SDK → TokenJoy Relay → 供应商 API
```

调用校验顺序：
1. Key 有效性
2. Key 启用状态
3. 模型白名单
4. 额度充足
5. 转发供应商

兼容格式：OpenAI API (chat completion) + Anthropic API (messages)

## 附录 B：数据模型

```
企业
├── 部门（树形结构）
│   ├── 成员
│   │   ├── 平台 Key
│   │   └── 个人额度
│   ├── 部门预算
│   ├── 虚拟项目组
│   └── 模型白名单规则
├── 角色
│   └── 权限点
├── 供应商 Key
├── 模型信息
├── 预警规则
├── 企业钱包
└── 审计日志
    ├── 操作日志
    └── 调用日志
```

### 核心实体

| 实体 | 主要属性 |
|------|----------|
| 企业 | 名称、数据源平台、总预算 |
| 部门 | 名称、父部门、来源（第三方/手动） |
| 成员 | 姓名、手机号、状态（启用/停用/未激活）、个人额度、已用额度 |
| 平台 Key | 名称、密钥前缀、所属成员、状态、额度、已用、绑定模型、过期时间 |
| 供应商 Key | 供应商、名称、状态、余额、是否自动轮转 |
| 角色 | 名称、类型（预设/自定义）、权限列表 |
| 预算节点 | 树形节点，总额、已分配、已使用、预留池 |
| 虚拟项目组 | 名称、额度、已用、所属部门、成员列表 |
| 预警规则 | 目标节点、阈值列表、通知角色、启用状态、目标类型 |
| 模型信息 | 供应商、名称、输入/输出价格、最大上下文、启用状态 |
| 路由规则 | 组织节点、允许模型、默认模型、备选模型、是否继承 |
| 审批单 | 类型（Key/额度）、申请人、状态（待审批/通过/拒绝）、申请额度 |
| 操作日志 | 操作类型、操作人、目标、详情、IP、时间 |
| 调用日志 | 调用人、模型、供应商、输入/输出 Token、延迟、状态、费用 |
