## 5. 已定案架构（实施时勿推翻）

| 问题                     | 决策                                              |
| ------------------------ | ------------------------------------------------- |
| 平行 `/enriched` API     | 否 — 扩展现有 `GET /keys/platform`、`GET /models` |
| `platform_keys` 推导字段 | 不入库；domain join enrich                        |
| 审批「我的申请」         | `memberId` 参数，不用 `tab=mine`                  |
| `visibility`             | P2 可编辑、**仅展示**；运行时 Phase 3             |
| UI 工程                  | `features/` + `use-*-page` + DI + 真 API          |

**`platform_keys` 字段分层**：`member_id` / `budget_group_id` / `app_name` 入库；`type`、`department_*`、`member_name`、`budget_group_name`、`project_name` 仅 JSON 响应。`relay_mappings.department_id` 为运行面快照，与管理面 enrich 分层不同。

**API 边界**：

```
GET /api/models         → ModelInfo[]（type / description / visibility / endpoint）
GET /api/keys/platform  → PageResult<PlatformKey>（departmentId、type 筛选 + enrich）
GET /api/keys/approvals → KeyApproval[]（tab × memberId）
```

## 8. 附录

### 8.1 生产迁移（`models` 四列）

```sql
ALTER TABLE models
  ADD COLUMN IF NOT EXISTS model_type   TEXT NOT NULL DEFAULT 'builtin',
  ADD COLUMN IF NOT EXISTS description  TEXT NOT NULL DEFAULT '',
  ADD COLUMN IF NOT EXISTS visibility   TEXT NOT NULL DEFAULT 'all',
  ADD COLUMN IF NOT EXISTS endpoint     TEXT;

UPDATE models SET model_type = 'custom' WHERE provider = 'custom' AND model_type = 'builtin';
UPDATE models SET model_type = 'builtin' WHERE provider <> 'custom' AND model_type = 'builtin';
UPDATE models SET visibility = 'all' WHERE visibility = '' OR visibility IS NULL;
```

本地开发：`docker compose down -v` 重建（见 [Backend-存储.md](./Backend-存储.md)）。
