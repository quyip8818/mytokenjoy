# ADR-001: 数据中心看板重设计

## 状态

已决策 — 2026-07-12

## 背景

当前"成本看板"和"用量分析"两个页面信息平铺，缺乏层次感。管理者无法在 30 秒内完成"AI 支出是否在控制范围内"的判断。

## 决策

### 用户分层

| 页面 | 核心用户 | 核心问题 |
|------|---------|----------|
| 成本看板 `/dashboard/cost` | CEO / CTO | "AI 支出在控制范围内吗？"（每周检查） |
| 用量分析 `/dashboard/usage` | 部门负责人 | "预算够吗？谁用得多？模型选对了吗？" |

### 成本看板改造

**Hero 区域（30 秒健康判断）：**
- 预算消耗率（进度条）："已消耗 ¥XX / 总预算 ¥XX（XX%）"
- 跟随左侧组织树选中节点
- 点击跳转预算管理详情页

**KPI 卡片精简：**
- 保留：总花费+环比、总调用次数+环比、人均成本+环比
- 去掉：总 Token（CEO 不关心 token 粒度）、平均单次成本（缺参照系）

**图表区域（下钻归因）：**
- 每日花费趋势折线图（保留）
- 部门成本占比饼图（保留）
- 子部门费用对比表（保留）
- 消费排行 Top N（保留，确保显示人名而非 ID）

**时间范围：**
- 新增"本周"选项
- 默认值保持"本月"

### 用量分析改造

**保留：**
- 团队用量与配额表（回答"预算够不够"）
- 模型费用分布图（回答"模型选对了吗"）

**新增：**
- 成员消耗排行表（回答"谁用得多谁用得少"）

**时间范围：**
- 新增"本周"选项
- 默认值保持"本月"

### 不做的事情

- 不做月底超支预测（线性外推容易误导）
- 不做"一句话归因"智能分析（当前保持简单）
- 不做点击趋势图/排行榜的深度下钻（组织树切换够用）
- 不做按角色切换布局（用组织树 scope 数据实现分层）

## 交互模型

下钻路径唯一：左侧组织树切换部门节点 → 页面数据自动 scope 到该节点。

## 技术约束

- 图表库：现有库够用，不引入新依赖
- 预算数据：从现有预算系统 API 获取，在成本看板做轻量摘要展示
- 后端 API：现有按 department × member × model 的聚合能力足够支撑
