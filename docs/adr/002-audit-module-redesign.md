# ADR-002: 审计模块优化

## 状态

已决策 — 2026-07-12

## 背景

审计模块当前只有纯表格形式，缺乏可视化能力和统计概览。核心使用场景为安全调查、合规审计和日常运维。

## 决策

### 用户场景

| 场景 | 页面 | 需求 |
|------|------|------|
| 安全调查 | 操作审计 | 快速定位某时间段内的异常操作 |
| 合规审计 | 操作审计 | 时间线视角回顾操作历史 |
| 日常运维 | 调用日志 | 错误率、延迟概览，快速发现异常调用 |

### 操作审计页面改造

**新增：按天时间轴**
- 位于筛选器和表格之间
- 按天分组的节点，hover 显示当天操作数
- 点击某天 → 自动将时间范围设为该天
- 展示范围跟随当前时间筛选器选择的范围

**筛选器改造：**
- 时间选择器从下拉框改为 DateRangePicker
- 快捷选项：今天、昨天、最近一周、最近一月
- 其他筛选器（操作类型、操作人、关键词）保持不变

### 调用日志页面改造

**新增：统计摘要卡片**
- 顶部 3 个 KPI 卡片：总调用数、错误率、平均延迟
- 数据来源：后端新增聚合接口

**筛选器改造：**
- 时间选择器从下拉框改为 DateRangePicker（同操作审计的快捷选项）
- 其他筛选器（状态、模型、调用人、关键词）保持不变

### 不做的事情

- 不做告警标注（异常行标红等 —— 后续再考虑）
- 不做主动推送（飞书/钉钉通知等）
- 不改调用日志的展开详情视图
- 不做调用日志的图表可视化

## 后端改动

需新增聚合接口：
- `GET /audit/calls/summary` — 返回指定时间范围内的总调用数、错误率、平均延迟

## 技术约束

- 前后端一起实现
- DateRangePicker 使用现有 shadcn/ui date picker 组件
- 时间轴使用 Recharts BarChart 或自定义 CSS 实现
- 聚合接口需支持与调用日志相同的筛选参数（时间范围、状态、模型、调用人）
