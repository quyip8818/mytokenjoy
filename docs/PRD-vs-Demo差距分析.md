# TokenJoy PRD vs Demo 差距分析（前端 Demo）

> 对比基准：[TokenJoy-PRD.md](./TokenJoy-PRD.md)  
> 交互基准：[Demo-交互设计方案.md](./Demo-交互设计方案.md)  
> 范围：**纯前端 Demo**（MSW Mock），评估 **P1～P3**（US-01～05、US-07～11）  
> 不含：P4 运营与合规（US-13～14）、US-12 API 网关、敏感词审查  
> 更新日期：2026-06-24

---

## 1. 总体结论

Demo 已覆盖 **P1 初始化 → P2 资源管控 → P3 成员 Key 闭环** 的主线演示路径，配合 [演示引导](../src/features/demo/guide/constants.ts) 可走通 8～10 分钟管理员脚本 + 成员脚本。

| 阶段          | 状态      | PRD 验收对齐度（UI/流程）                 |
| ------------- | --------- | ----------------------------------------- |
| P1 平台初始化 | ✅ 较完整 | 约 85～95%                                |
| P2 资源管控   | 🟡 简版   | 约 70～85%（配置可走通，运行时多为 Mock） |
| P3 成员接入   | ✅ 较完整 | 约 85～90%                                |

**路演可表述**：覆盖 PRD P1～P3 **业务主线**；**不可**声称满足 PRD 全部验收标准（定时同步、月重置、运行时预警/阻断、API 网关等）。

## 2. Demo 覆盖范围声明

### 2.1 支持（可演示核心路径）

| US    | 名称         | Demo 入口 / Workflow                                               |
| ----- | ------------ | ------------------------------------------------------------------ |
| US-01 | 第三方凭证   | `/org/data-source` → `credential-form`                             |
| US-02 | 全量导入     | 数据源全量导入 + `import-result` 重试                              |
| US-03 | 定时同步     | `sync-config` + 同步日志 + 手动触发                                |
| US-04 | 组织架构     | `/org/structure` + `member-*` / `dept-form`                        |
| US-05 | 角色权限     | `/org/roles` + `role-form`（Demo 三角色切换，非真实 RBAC）         |
| US-07 | 逐级预算     | `/budget/overview` + `budget-node-edit`；BG → `/budget/allocation` |
| US-08 | 超限策略     | `/budget/alerts` + `overrun-policy`（全局配置）                    |
| US-09 | 模型白名单   | `/models/list` + `/models/routing` + `whitelist-config`            |
| US-10 | 审批流       | `approval-submit` / `approval-review` / `/keys/approval`           |
| US-11 | 自主管理 Key | `/keys/mine` + `key-create` / `key-edit` 等                        |

### 2.2 简版（有 UI，规则为 Mock 或未全量验收）

| 项                               | 说明                                            |
| -------------------------------- | ----------------------------------------------- |
| US-03 定时触发                   | 无真实 cron；同步日志含**超阈值终止**样例供讲解 |
| US-03 本地手动部门免疫           | 业务规则未在 UI 可验证                          |
| US-04 成员筛选                   | 无「仅直属 / 全部」切换                         |
| US-05 五类预设角色               | Mock 数据有；Demo 切换器仅 admin / tl / member  |
| US-07 自然月重置                 | 预算页脚注说明由后端处理，Demo 不模拟           |
| US-07 BG 独立 API Key            | 未实现                                          |
| US-08 运行时预警/阻断            | 仅配置页；阻断依赖 US-12                        |
| US-09 API 403 模型错误           | 无 API 层演示                                   |
| US-10 IM 通知                    | Toast/Mock                                      |
| US-11 单 Key 额度用尽 / API 调用 | 无 US-12 网关                                   |

### 2.3 不支持（范围外）

| 项                     | 原因                                       |
| ---------------------- | ------------------------------------------ |
| US-12 API 调用         | 纯 API 行为，无 UI                         |
| US-13 成本看板完整验收 | P4；页面只读展示存在                       |
| US-14 审计完整验收     | P4；导出、内容留存开关、只读审计员视角未齐 |
| 敏感词审查             | PRD 价值主张提及，无独立 US                |
| 真实 RBAC              | 设计文档 §17.5 延后                        |

## 7. 演示走查清单

演示前建议按 [DEMO_GUIDE_STEPS](../src/features/demo/guide/constants.ts) 执行，并确认：

- [ ] US-01：切换平台二次确认、测试连接前不可保存
- [ ] US-02：导入失败可单条/批量重试
- [ ] US-03：同步日志可见超阈值终止样例
- [ ] US-04：停用成员后其 Platform Key 变禁用
- [ ] US-07：预算超卖被 Banner 阻断
- [ ] US-09：子部门白名单仅能选父级子集
- [ ] US-10：额度追加预留池不足 → `quota-check`；Key 审批通过 → 申请人名下出现新 Key
- [ ] US-11：剩余额度为 0 时无法创建 Key；创建成功 → `key-reveal` → 行高亮
