# TokenJoy PRD vs Demo 差距分析（前端 Demo）

> 对比基准：[TokenJoy-PRD.md](./TokenJoy-PRD.md)  
> 交互基准：[Demo-交互设计方案.md](./Demo-交互设计方案.md)  
> 范围：**纯前端 Demo**（MSW Mock），评估 **P1～P4**（US-01～14）  
> 不含：US-12 API 网关、敏感词审查配置、真实后端  
> 更新日期：2026-06-25

---

## 1. 总体结论

Demo 已覆盖 **P1 初始化 → P2 资源管控 → P3 成员 Key 闭环** 的主线演示路径，配合 [演示引导](../apps/frontend/src/features/demo/guide/constants.ts) 可走通 8～10 分钟管理员脚本 + 成员脚本。P4 看板/审计为**简版可演示**。

| 阶段          | 状态      | PRD 验收对齐度（UI/流程）            |
| ------------- | --------- | ------------------------------------ |
| P1 平台初始化 | ✅ 较完整 | 约 85～90%                           |
| P2 资源管控   | 🟡 简版   | 约 75～85%（个人额度池已对齐 Mock）  |
| P3 成员接入   | ✅ 较完整 | 约 90～95%                           |
| P4 运营合规   | 🟡 简版   | 约 55～65%（时间切换/下钻/导出已补） |

**路演可表述**：覆盖 PRD P1～P3 **业务主线** + P4 **基础看板/审计**；**不可**声称满足 PRD 全部验收标准（定时同步 cron、真实 API 网关、完整审计留存策略等）。

## 2. Demo 覆盖范围声明

### 2.1 支持（可演示核心路径）

| US    | 名称         | Demo 入口 / Workflow                                               |
| ----- | ------------ | ------------------------------------------------------------------ |
| US-01 | 第三方凭证   | `/org/data-source` → `credential-form`                             |
| US-02 | 全量导入     | 数据源全量导入 + `import-result` 重试                              |
| US-03 | 定时同步     | `sync-config` + 同步日志 + 手动触发                                |
| US-04 | 组织架构     | `/org/structure` + `member-*` / `dept-form`；**仅直属/全部**筛选   |
| US-05 | 角色权限     | `/org/roles` + `role-form`；**5 角色**切换器 + 删除二次确认        |
| US-07 | 逐级预算     | `/budget/overview` + `budget-node-edit`；BG → `/budget/allocation` |
| US-08 | 超限策略     | `/budget/alerts` + `overrun-policy`（全局配置）                    |
| US-09 | 模型白名单   | `/models/list` + `/models/routing` + `whitelist-config`            |
| US-10 | 审批流       | `approval-submit` / `approval-review` / `/keys/approval`           |
| US-11 | 自主管理 Key | `/keys/mine` + `key-create` / `key-edit` 等                        |
| US-13 | 成本看板     | `/dashboard/cost` — 时间维度切换 + 部门下钻                        |
| US-14 | 审计追踪     | `/audit/*` — 筛选、导出、内容留存开关                              |

### 2.2 简版（有 UI，规则为 Mock 或未全量验收）

| 项                               | 说明                                            |
| -------------------------------- | ----------------------------------------------- |
| US-03 定时触发                   | 无真实 cron；同步日志含**超阈值终止**样例供讲解 |
| US-03 本地手动部门免疫           | 业务规则未在 UI 可验证                          |
| US-04 成员激活完整路径           | 邀请可发，无成员自助注册激活流程                |
| US-05 真实 RBAC                  | 权限静态映射，非后端动态下发                    |
| US-07 自然月重置                 | 预算页脚注说明，Demo 不模拟月初清零             |
| US-07 BG 独立 API Key            | 未实现                                          |
| US-07 TL 配置平均额度/人 UI      | 个人池在 Mock 中维护，无组织页配置入口          |
| US-08 运行时预警/阻断            | 仅配置页；阻断依赖 US-12                        |
| US-09 API 403 模型错误           | 无 API 层演示                                   |
| US-10 IM 通知                    | Toast/Mock                                      |
| US-11 单 Key 额度用尽 / API 调用 | 无 US-12 网关                                   |
| US-13 自定义时间范围             | 仅预设：本月/上月/近7天                         |
| US-14 操作审计详情 Panel         | 导出已实现；操作快照详情仍为表格截断            |
| US-14 热存储/归档策略            | 未体现                                          |

### 2.3 不支持（范围外）

| 项             | 原因                                  |
| -------------- | ------------------------------------- |
| US-12 API 调用 | 纯 API 行为，无 UI / 无网关           |
| 敏感词审查     | PRD 价值主张提及，无独立 US 与配置 UI |
| 租户创建/部署  | Demo 直接进入已存在租户               |
| 真实登录 SSO   | Header 角色切换器模拟身份             |

## 7. 演示走查清单

演示前建议按 [DEMO_GUIDE_STEPS](../apps/frontend/src/features/demo/guide/constants.ts) 执行，并确认：

- [ ] US-01：切换平台二次确认、测试连接前不可保存
- [ ] US-02：导入失败可单条/批量重试
- [ ] US-03：同步日志可见超阈值终止样例
- [ ] US-04：停用成员后其 Platform Key 变禁用；直属/全部筛选可用
- [ ] US-07：预算超卖被 Banner 阻断；删除 Key 后个人剩余额度增加
- [ ] US-09：子部门白名单仅能选父级子集
- [ ] US-10：额度追加预留池不足 → `quota-check`；超白名单模型无法提交
- [ ] US-11：剩余可分配额度为 0 时无法创建 Key；创建成功 → `key-reveal`
- [ ] US-13：成本看板切换「近7天」；部门下钻至成员
- [ ] US-14：关闭内容留存后调用日志不展示 prompt/response；导出 CSV
