# 预警规则页面优化方案

## 现状分析

当前页面结构简单：顶部操作栏 + 纯表格列表。存在以下体验问题：

1. **无概览信息** — 管理员无法快速感知预警规则覆盖情况（多少规则已启用、覆盖了多少部门/项目）
2. **空状态单调** — 仅显示文字"暂无预警规则"，缺乏引导性
3. **信息密度过高** — 7 列表格在窄屏下拥挤，"通知角色"仅显示数量缺少直观感
4. **缺少筛选** — 规则多时无法按类型（团队/项目）、状态（启用/禁用）过滤
5. **缺少批量操作** — 无法快速批量启用/禁用

## 优化方案

### 1. 顶部统计卡片区

在操作栏下方、表格上方增加一组 stat cards，提供规则全景：

```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  规则总数     │  │  已启用       │  │  团队覆盖     │  │  项目覆盖     │
│     12       │  │     9        │  │    5/8       │  │   7/15       │
└──────────────┘  └──────────────┘  └──────────────┘  └──────────────┘
```

- 从现有 `rules` 数据计算，不需要新接口
- 团队/项目覆盖率 = 有规则的唯一团队数 / 总团队数

### 2. 筛选工具栏

在统计卡片与表格之间加入筛选区：

```
[ 全部 | 团队 | 项目 ]    [ 全部状态 ▾ ]    [ 搜索监控对象... 🔍 ]
```

- **类型 Tab 切换**：全部 / 团队 / 项目（参考 platform-keys-toolbar 的 Tab 风格）
- **状态下拉**：全部 / 已启用 / 已禁用
- **搜索框**：按监控对象名称模糊过滤

### 3. 表格优化

| 优化项 | 说明 |
|--------|------|
| 合并"类型"到"监控对象"列 | 用小 Badge 跟在名称后面，减少一列 |
| "通知角色"展开显示 | 最多显示 2 个角色名 + `+N`，hover tooltip 显示全部 |
| "动作"列用图标+文字 | 增加辨识度（🚫硬拒绝 / ✅审批追加 / ⚡降级路由） |
| 行内启用/禁用 | 保留 Switch，但禁用时整行降低透明度视觉区分 |
| 操作列收入 DropdownMenu | 编辑/删除/复制规则，减少视觉噪声 |

优化后表格列：`监控对象` | `阈值` | `触发动作` | `通知角色` | `状态` | `操作`（6列→更宽敞）

### 4. 空状态优化

使用项目已有的 empty state 模式（参考 `FilteredPageShell` 的 `empty` prop）：

- 居中大图标 `Bell` + 标题"尚未配置预警规则"
- 描述文字"设置预警阈值，在预算即将超支时及时通知相关负责人"
- 行动按钮"创建第一条规则"

### 5. 禁用规则视觉弱化

禁用的规则行添加 `opacity-50`，让管理员一眼区分哪些是活跃的。

## 实现计划

### 文件改动

1. **新增** `budget-alerts-stats.tsx` — 统计卡片组件
2. **新增** `budget-alerts-toolbar.tsx` — 筛选工具栏组件
3. **修改** `budget-alerts-table.tsx` — 表格列优化 + 禁用行弱化 + DropdownMenu
4. **修改** `budget-alerts-page-shell.tsx` — 组装新组件、接入筛选状态
5. **修改** `use-budget-alert-rules-page.ts` — 新增筛选 state + 派生计算 stats

### 不涉及的改动

- 无后端接口变动（所有数据前端已有）
- 无新依赖引入
- 无路由变动
