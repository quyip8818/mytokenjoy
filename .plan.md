# Plan: 将 UI 从 Base UI 切换到 Shadcn/ui

## 概要

移除 @base-ui/react，引入 shadcn/ui（基于 Radix + Tailwind），用 shadcn 组件重写所有页面中的 UI 原语（button、input、select、dialog、switch、table、checkbox 等）。

## 步骤

### 1. 配置 shadcn/ui 基础设施

- tsconfig.json / tsconfig.app.json 添加 `@/*` 路径别名
- vite.config.ts 添加 resolve.alias
- 运行 `npx shadcn@latest init` 初始化（生成 components.json + src/lib/utils.ts 覆盖）
- 卸载 `@base-ui/react`

### 2. 安装需要的 shadcn 组件

根据当前页面用到的原语，添加以下 shadcn 组件：

- button, input, label, select, switch, checkbox
- dialog, alert-dialog (替代 confirm-dialog)
- table (样式化 table，配合 TanStack Table)
- dropdown-menu (批量操作、更多操作)
- badge (角色标签、状态标签)
- toast / sonner (通知)
- card (页面区块)
- separator
- radio-group

### 3. 重写共享组件

- 删除 `src/components/shared/confirm-dialog.tsx`，用 shadcn 的 AlertDialog 替代
- 更新 `src/components/ui/` 目录（shadcn 组件自动生成到这里）

### 4. 重写业务页面（并行）

- 数据源页面 (credential-form, import-result, sync-config, sync-log-table, data-source)
- 组织架构页面 (department-tree, member-table, member-form, structure)
- 角色管理页面 (role-list, role-form, role-member-table, roles)
- Layout (sidebar, header, admin-layout)

每个文件中的原生 `<button>`, `<input>`, `<select>`, `<dialog>` 替换为对应的 shadcn 组件导入。

### 5. 清理

- 删除 index.css 中的 dialog 居中 hack（shadcn Dialog 自带居中）
- 确认构建通过
